### INTERFACE ARCHITECTURE & TECHNICAL BLUEPRINT
Whenever generating layouts, components, or interactive logic, you must strictly inherit the following front-end engineering blueprint:

1. Layout & Architecture Patterns:
   - Architecture: Clean Single Page Application (SPA) configurations using fluid horizontal stage transitions driven by smooth viewport transforms (`transform: translateX()`).
   - Layering & Depth: Use modern, fluid, scale-shifting ambient blurred background blobs animating indefinitely on relaxed loops (`@keyframes bgFloat`) to establish depth without relying on legacy blurry drop shadows.

2. Interaction Engineering & Micro-animations:
   - Fluid Component Morphing: Premium interactive elements, images, or cards must utilize dynamic, continuous fluid shape-shifting properties driven by multiple transitioning border-radiuses (`@keyframes shapeMorph`).
   - Magnetic Interactivity: Focus components or call-to-actions must hook into custom JavaScript-driven cursor tracking, implementing precise matrix transformations to pull elements towards mouse coordinates dynamically.
   - Sub-pixel Point-of-Click Ripples: Every clickable surface must embed highly responsive Material ripple effects (`.m3-ripple-wave`) calculated relative to the exact point-of-click coordinate on the element canvas.

#### 2. Geometry & Shapes Scale
Exaggerated shapes and structural asymmetry direct user focus.
* Shape Tokens:
  - `--md-shape-corner-small`: 12px (Chips, contextual inputs, micro-data badges)
  - `--md-shape-corner-medium`: 16px (Standard content tiles, component blocks, pop-ups)
  - `--md-shape-corner-large`: 28px (Main feature cards, parent dashboard panels, window states)
  - `--md-shape-corner-full`: 9999px (Pill-shape main CTAs, active indicators, floating background blobs)
* Asymmetric Expressive Accents: For primary layouts or focus points, implement intentional multi-radius properties combining contrasting sharp and large radiuses on a single container surface.

#### 3. Typography Matrix
Type acts as an active graphic overlay. Use clean, variable sans-serif choices (e.g., "Plus Jakarta Sans", "Inter") with high tracking and weight differentiation.
* Display / Headlines: Bold, heavy weights (800+ or ExtraBold/Black) with condensed letter-spacing (`letter-spacing: -0.03em`) and crisp, tight line-heights (1.15) to drive maximum structural impact.
* Body / UI Labels: Meticulously balanced, clean weights (400–500) leveraging high text-to-background contrast across both theme states.

#### 4. Motion, Elasticity & Fluid Transitions
Interface components must feel physically alive, snappy, and responsive.
* Physics Easing: All transitions must map to the signature Emphasized curve:
  - `--md-easing-emphasized`: cubic-bezier(0.2, 0, 0, 1)
