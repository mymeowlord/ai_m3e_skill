#### 2. Geometry & Shape Scale Matrix
Expressive layouts employ structured asymmetry and exaggerated corner radiuses to convey brand character and direct user focus.
* Shape Tokens:
  - `--md-shape-corner-none`: 0dp (Edge-to-edge screens, full-bleed media structures)
  - `--md-shape-corner-extra-small`: 4dp (Status micro-indicators, small utility icons)
  - `--md-shape-corner-small`: 12dp (Chips, contextual inputs, micro-data badges, tooltips)
  - `--md-shape-corner-medium`: 16px (Standard content tiles, interactive component blocks, dialogs)
  - `--md-shape-corner-large`: 28px (Main feature cards, parent dashboard panels, window states)
  - `--md-shape-corner-extra-large`: 32dp (Floating Action Buttons, prominent marketing banners)
  - `--md-shape-corner-full`: 9999px (Pill-shape main CTAs, active indicators, floating background blobs)
* Asymmetric Expressive Accents: For critical tiles or brand promotional surfaces, apply specialized multi-radius combinations (e.g., combining sharp corners with Extra-Large radiuses on opposite bounds of a single container canvas).

#### 3. Typography & Variable Font Hierarchy
Type functions as an active graphical element. You must leverage clean, variable sans-serif choices (e.g., "Plus Jakarta Sans", "Inter") using dynamic tracking scales.
* Display / Headlines: Bold, expressive, heavy weights (800+ or ExtraBold/Black) with condensed letter-spacing (`letter-spacing: -0.03em`) and crisp, tight line-heights (1.15) to drive maximum structural impact.
* Body / UI Labels: Meticulously balanced, clean weights (400–500) leveraging high text-to-background contrast across both theme states.

#### 4. Motion, Elasticity & Fluid Transitions
Every interface component must respond organically, mimicking real physical elasticity and fluid motion principles.
* Official Physics Easing: All transition timing architectures must map to the official signature Emphasized curve:
  - `--md-easing-emphasized`: cubic-bezier(0.2, 0, 0, 1)
  - `--md-duration-transition`: 0.5s
* Core Micro-interactions:
  - Viewport Stage Transitions: Use smooth layout shifting using container transforms and precise translation states (`transform: translateX()`).
  - Fluid Component Morphing: Interactive focus points or image blocks use shifting border-radiuses with continuous structural fluid transitions (`@keyframes shapeMorph`).
  - Point-of-Click Ripples: Clickable inputs or controls must embed highly responsive sub-pixel point-of-interaction ripple waves (`.m3-ripple-wave`) calculated relative to the exact coordinate of the user's action.
