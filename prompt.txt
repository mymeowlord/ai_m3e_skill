### REFERENCE GUIDE: MATERIAL YOU 3 EXPRESSIVE SPECIFICATION

#### 1. Design Philosophy & Core Vision
Material 3 Expressive is an evolution of classic M3, shifting the paradigm from rigid utility to high-fidelity brand personalization, dynamic scaling, and deep emotional resonance. The goal is to move away from generic "out-of-the-box" Android templates toward highly customized geometry, striking variable typography, and fluid, organic motion.
Key Style Markers:
* Bold Geometry: High-contrast combinations of extreme corner radiuses and sharp structural edges.
* Brand-Centric Dynamic Color: Adaptive color scaling driven by user choices or strict brand identity palettes (UGC/Brand Centric).
* Scale & Whitespace: Generous padding, oversized headers, and an absolute focus on content immersion.

#### 2. Color Architecture & Tonal Palettes (Color System)
The interface is built entirely on a 65-tonal palette system. Dynamic theme generation (Light/Dark mode) must strictly adhere to WCAG AAA accessibility standards (minimum 4.5:1 contrast ratio for regular text).
Mandatory Semantic Tokens:
* Primary / On-Primary: The core brand identifier. Reserved strictly for critical Call-to-Actions (CTAs).
* Secondary / On-Secondary: Subdued tonal variants for utility elements (chips, filters, tags).
* Tertiary / On-Tertiary: Critical for the Expressive style. A high-contrast accent color used to disrupt visual monotony, highlight special offers, or anchor notifications.
* Surface Container: Replaces legacy elevation shadows. Layering is achieved via color depth hierarchy (from Lowest to Highest), rather than drop shadows.
* Rule: Hardcoded color values (HEX/RGB) are strictly prohibited. All elements must map to system design tokens (e.g., `md.sys.color.primary`).

#### 3. Geometry, Shapes & Corner Radiuses
Expressive design embraces asymmetry and exaggerated corner radiuses to inject personality into layout hierarchies.
Shape Scale Matrix:
* None (0dp): Edge-to-edge screen components, full-bleed imagery.
* Extra Small (4dp): Micro-components, inline indicators, small utility badges.
* Small (8dp): Input fields (TextField), tooltips, contextual dropdowns.
* Medium (12dp): Standard product cards, minor dialog boxes.
* Large (16dp–24dp): Major container layouts, standard Bottom Sheets.
* Extra Large (28dp–32dp): High-impact promo banners, marketing cards, Floating Action Buttons (FAB).
* Full (100% / Pill-shape): Interactive buttons, chips, search bars.
* Expressive Accent Edge: For promotional elements, you may apply asymmetric corner radiuses on a single container (e.g., top-left and bottom-right at 32dp, while top-right and bottom-left remain at 12dp).

#### 4. Typography Hierarchy
Typography functions as a primary graphical element. We utilize Variable Fonts to dynamically manipulate font weight and tracking based on context.
* Display (Large, Medium, Small): Splash screens, hero sections, and key metrics. In Expressive, these use UltraBold / Black weights with tightened letter-spacing (tracking).
* Headline (Large, Medium, Small): High-contrast, sharp section headers.
* Title (Large, Medium, Small): Card titles, list headers, and prominent UI labels.
* Body (Large, Medium, Small): Main reading text. Prioritizes extreme legibility, balanced line height, and standard tracking.
* Label (Large, Medium, Small): Button text, icon captions, and micro-technical metadata.

#### 5. Expressive UI Components
Standard components must be refactored to meet Expressive guidelines:
* Buttons: Main CTA heights are increased to 48dp–56dp utilizing a full Pill-shape. Interaction triggers a vivid, highly visible Ripple effect with enhanced base opacity.
* Text Fields (TextField): Filled container types utilizing a soft Surface Container background. The active indicator line scales up to 2dp upon focus, shifting to the Primary token color.
* Cards: Zero heavy, blurry drop shadows. Separation is achieved through semantic background contrast (e.g., a Surface Container High card placed over a Surface background). Outlines use a crisp 1dp border mapped to Outline Variant.
* Navigation Bar / Rail: Active states are highlighted using a distinct pill-shaped indicator. The accompanying text scales smoothly or fades in gracefully upon activation.

#### 6. Motion, Transitions & Micro-interactions
Motion is an informational guide, not just decoration. It must feel organic, responsive, and elastic.
* Emphasized Easing: Transitions must utilize the Emphasized easing curve (rapid acceleration with a long, smooth deceleration ease-out). Formula: `cubic-bezier(0.2, 0.0, 0, 1.0)`
* Container Transform: When a user interacts with a card, it physically morphs and expands into the new screen, smoothly transitioning its corner radius from 24dp to 0dp as it fills the viewport.
* Parallax Depth: Foreground elements (like a rising Bottom Sheet) must move at a different velocity than the background layer to establish true visual depth.

#### 7. Technical Implementation Requirements (Dev Handoff)
* Android (Jetpack Compose): Implement by overriding `MaterialTheme`. Custom design parameters for colors, shapes, and typography must be fed directly through the project's customized token system.
* iOS (SwiftUI): Establish an identical mirrored architecture using native Color, Font, and CornerRadius structures that precisely map back to the M3 Design Tokens logic.
* Responsiveness (Window Size Classes): Seamless scaling across Compact (phones), Medium (foldables/mini-tablets), and Expanded (tablets). On larger viewports, the grid transitions to a multi-column layout with padding scaling up to 24dp–32dp.
* Dark Mode Support: Color inversion must strictly follow the tonal palette grid (e.g., Light Tone 90 maps directly to Dark Tone 10). Pure black backgrounds (`#000000`) are strictly forbidden; use deep dark hues via `Surface` / `Surface Container`.
